## §2 🧭 信息层级 / 阅读顺序：改完更乱了

### 2.1 三选一小胶囊是摆设吗？

`L136-152` 一排"🎴 Show / 🔍 Scout / ⚡ S&S"三色胶囊，每个里一个 emoji + 一行 10px 小字。看完之后紧接着 `L154-231` 又是 Show / Scout / S&S 的详细段落。

> 我读了两遍上面那排色块的意义是啥？**占屏幕 1/3 宽度，信息量 = 下面段落前两个字。** 装饰画吗？要么砍掉，要么把它做成「点击跳转到对应详细段」的锚点，否则就是让我滚两屏读两次。

### 2.2 Show 段嵌套了一层"强度比较框"，Scout / S&S 没有

`L174-203` 的"牌型强度比较顺序"其实是 Show 的**子规则**，但它塞了一整个灰底大框 + 4 组卡示意图 + caption。结果是：
- Show → 正文 → 子框（带图）→ 继续看
- Scout → 正文一句话
- S&S  → 正文一句话

**三个兄弟，老大穿大棉袄老二老三穿汗衫**。视觉层级完全不对等，我读完 Show 以为后面两个会更详细，结果一闪而过，"就这？"——害我又回头怀疑是不是漏读了。

### 2.3 翻面机制被切成三块，没有总览

- `L115-116`「卡片速览」里说：翻面后两数互换，颜色从青变琥珀。
- `L213`「Scout」里说：可选择是否翻面后再插入手牌。
- `L276-287`「🔁 整副翻转」又单独开了一段。

**翻面这事被我读到了三次，每次都是半句话，从来没有人告诉我：**
> "本游戏有三种翻面：① 单张 Scout 时可翻；② 每轮开始一次整副翻；③ 翻面只改变朝上数字。"

新手脑子里会混成一锅："所以 Scout 翻的是整副还是单张？整副翻之后又能 Scout 再翻吗？翻完还能翻回来吗？"——全是疑问。

### 2.4 "③ 一轮何时结束"放最下面，但计分要靠它收尾

`L266-273` 的结束条件里才第一次说清楚**剩牌扣分到底谁扣谁不扣**（Show 者免扣）。但我读 ② 计分的时候就已经看到了"−剩牌"胶囊（`L253-258`），当时以为"所有人都按剩牌数扣"。等到 ③ 才补一句"条件 ② 下 Show 者免扣"——

> **计分规则被拆到了两个地方讲，中间隔了一整个"结束条件"段。** 让我在脑子里拼完整规则我是不是得写草稿纸？

### 2.5 默认展开顺序反了：规则在前、例子在后且折叠

暴躁新手学新游戏的正确顺序是 **先看一个完整例子建立直觉，再回头抠规则**。现在是：
- `QuickRulesCard` 默认 **展开** → 300 行规则糊我脸上
- `RoundExampleCard` 默认 **折叠** → 还是灰字 11px 副标题"完整回合走一遍"

我读规则读到一半已经想退出了，**压根没注意到下面还有个例子卡可以展开**。建议反过来：例子卡默认展开放最前，规则卡做参考放后面。

## §3 🗣 老玩家口吻 / 术语假设：黑话依然不少

### 3.1 "盖过场上当前牌组"（`L166`）——"盖过"是啥？

读到这里我一脸懵。盖过是物理上盖上去？数字上更大？"盖过"的条件是什么？
**答案藏在后面那个强度比较框里**，但"盖过"这个动词在它第一次出现的时候就没有任何解释。

> 建议在第一次出现时加个括号："盖过（= 比场上牌组更强，强度规则见下）"。

### 3.2 "桌面无牌时可任意出"（`L166`）—— 一个"任意"吞掉了所有细节

新手会问：
- 任意多少张？
- 任意什么牌型？还必须符合"同数字/连续"吗？
- 能一次甩 10 张开局？

**"任意"在规则里是禁用词**。写规则的人心里有数，读规则的人两眼一黑。

### 3.3 "仍标记为你的"和"+吃牌"没对上暗号

`L169` Show 段说："打出的牌叠到场中央，仍**标记为你的**——对手 Scout 走时你得分（见计分）"。
`L241` 计分段说："被盖过的那组牌**收到自己面前**。每张 = 1 分。"

所以——
- 我 Show 出的牌叠"场中央"但"仍是我的"
- 被别人 Show 盖过后的牌进"自己面前"算+1 分

**两件事的主语是一个概念吗？** 等等，"收到自己面前"里的"自己"是新主人还是原主人？读三遍才捋清楚是"盖过别人的人"吃掉被盖的牌。但之前一直说"仍是原主人的"……**所以所有权什么时候转移的？** 没！说！

### 3.4 位置名词五连杀

全篇对"那堆牌放哪"的称呼：

| 称呼 | 出现位置 |
|------|---------|
| 桌面 | `L166`「桌面无牌时」、`L334`「桌面 Scout」 |
| 场上 | `L163`「盖过场上」、`L212`「场上牌组」、`L269`…… |
| 场中央 | `L169`「叠到场中央」 |
| 自己面前 | `L241`「收到自己面前」 |
| 场上牌组 | `L212` |

**同一个东西五个词**。新手读完怀疑人生："这游戏难道有五个不同的区域？"——请统一成一个名词，最多两个（比如「场上 / 你的得分堆」）。

### 3.5 "轮"vs"回合"继续混用

v1 吐过，这版**没改**：
- `L227` S&S："整轮限 1 次"
- `L266` "一轮何时结束"
- `L282` "每轮限 1 次"

一轮到底是"4 人各出一次一圈"，还是"整局 4 轮里的 1 轮"？暴躁新手在这里已经要掀桌。

> 建议建立术语小词典：**一局 = 4 轮；一轮 = 从发牌到有人出空/全员 Scout；回合 = 一个人的一次行动**。并且整页严格使用这三个词，不要再冒出"整轮""每轮"这种模棱两可的修饰。

### 3.6 "被抽"计分主谓链过长

`L248-249`："对手从你 Show 出的牌组里 Scout 走 1 张时，你（原主人）得 1 分。"

读三遍才懂。建议主谓直给：
> 「**你 Show 出的牌每被对手 Scout 走一张，你 +1 分。**」

### 3.7 "轮回到 Show 者"依然是黑话

`L268-269`："某人 Show 后，其余 3 人全部只选 Scout，轮回到 Show 者。"

**"轮回到 Show 者"是什么法术？** 白话版：
> 「你 Show 之后，其他三个人全部选 Scout 不出牌，一圈之后又转回到你——此时本轮结束。」

现在的表述像是从规则书里机翻过来的。

### 3.8 RoundExample 里"同数字"用在单张上很奇怪

`L345-346`："C 用 S&S：先从桌面 Scout 走 4，再立即 Show 出自己的 **6**（1 张同数字更大）盖过。"

**一张牌哪来的"同数字"？** 同数字不是起码两张相同才能叫吗？单张就直接说"打出一张 6，比场上的 4 大"就行。（但这里还有个更严重的 bug，见 §4.1）

## §4 🎨 视觉 / 排版：一屏 10 种颜色，眼睛没处落

### 4.1 配色过载

随手数一下这页用到的颜色：
- 琥珀黄警告条（`L131`）
- 青色胶囊（`L137`）
- primary 紫（`L143`）
- accent 橙（`L147`）
- 白色边框灰底方块（到处都是）
- `ink-50 / 100 / 200 / 300 / 400 / 500` 六档文字灰
- 青色强调色 `text-cyan-300`（随机撒在关键词上）

**一屏扫下来 10+ 种颜色**，我眼睛完全定位不到"哪个是重点"。警告条是黄的、强度框是白的、计分胶囊又是白的、结束条件是灰的——**所有东西都在抢 C 位，结果没有一个是 C 位**。

### 4.2 三选一胶囊信息密度过低（再说一遍）

- 每个色块：1 个 emoji + 1 个英文单词 + 1 行 10px 中文
- 占用：屏宽 1/3 × 3 块 = 整整一行

**提供的信息量 ≈ 下一段正文的第一句**。这就是装饰性排版的典型：**视觉很满、信息很空**。

### 4.3 Show / Scout / S&S 胶囊宽度对不齐

`L159` "Show（出牌）" vs `L208` "Scout（抽牌）" vs `L222` "S&S" —— 三个胶囊里文字长度不一，宽度也就不一，导致右侧正文的左缘在三段之间**跳来跳去**。扫读时视觉动线不稳，看完一行要重新定位下一行的起点。

### 4.4 强度比较图小屏易挤爆

`L184-202` 四组小卡并排 + 每组下面 caption（"3 张连续·张数最多"、"2 张同数字·强于连续"、"2 张连续"、"单张·最弱"）+ `›` 分隔符。

- `size="xxs"` 的卡已经非常小
- caption 12px 挤在下方
- 手机小屏（iPhone SE 375px）上四组 + 三个分隔符容易挤换行

而且并列展示**看不出递进**——四组看起来是"四种牌型"，而不是"从强到弱"。`›` 太弱，建议用更明确的视觉：比如从左到右色度渐弱 / 加「强 ← 弱」两端标签 / 或者直接用数字 ① ② ③ ④ 排序。

### 4.5 计分胶囊 + − 混排无色差

`L237 / L245 / L253` 三个胶囊：
- `+ 吃牌`（白边框、白灰底）
- `+ 被抽`（白边框、白灰底）
- `− 剩牌`（白边框、白灰底）

**加减号混排且颜色完全一致**。这是会涨分还是扣分？对新手来说"加号 = 好事、减号 = 坏事"的色彩暗示是刚需——绿色加分、红色扣分，最起码也要用两种色区分。

现在这样一眼看上去三条一样，读文字才知道是扣分——**暴躁新手只扫颜色，根本不读字**。

### 4.6 计分公式术语不对齐

`L261-263`："本轮得分 = **收牌数** + **被抽分** − **剩余手牌数**"

对比上面三个胶囊：
- 胶囊 `+吃牌` → 公式 `收牌数`（吃 ≠ 收）
- 胶囊 `+被抽` → 公式 `被抽分`（对上了）
- 胶囊 `−剩牌` → 公式 `剩余手牌数`（对上了）

**"吃牌"和"收牌数"是同一件事吗？** 99% 是，但你不能让我猜。建议胶囊和公式**用同一个词**，或者公式直接引用胶囊名："本轮得分 = 吃牌数 + 被抽数 − 剩牌数"。

### 4.7 RoundExample 折叠入口太弱

`L311-312` 标题"🎬 看个例子"是 14px 粗体，副标"完整回合走一遍"是 11px 灰字（`text-ink-400`）。

在一屏全是色块、警告条、胶囊的环境里，**一个平平无奇的黑底灰字按钮根本抢不到视线**。如果它承担"新手最该先看的内容"，至少：
- 副标改成 primary 色或 accent 色
- 或加个小红点 / NEW 标签
- 或者干脆默认展开（见 §2.5）

### 4.8 底部按钮"知道了"依然冷淡

v1 建议过改成 **"我懂了，开打！"** 或 **"还是不懂，看视频教程 →"**。这版照旧是"知道了"。

> 对一个暴躁读完规则的新手来说，"知道了"三个字像极了家长训完话的收尾。能不能多点情绪价值。

## §5 自相矛盾 / 硬 bug

### 5.1 🐛 RoundExample 里 C 到底 Show 了一张还是两张 6？

`L345-346`：
> C 用 S&S：先从桌面 Scout 走 4，再立即 Show 出自己的 **6**（1 张同数字更大）盖过。

**但紧接着 `L347`：**
> A 再因被抽 1 张得 +1 分；C 吃掉桌上剩下的 3（+1 分）。

然后 `L350` 第 4 步：
> D 选择 Show：打出 **7-7** 盖过 C 的 **6-6**（同 2 张同数字但更大）。

`L345` 明明说 C 只 Show 了 **1 张 6**，到 `L350` 突然变成 **6-6（2 张）**——**自相矛盾**。要么把 `L345` 改成"Show 出 6-6"并说明 C 手里有两张 6，要么把 `L350` 的 D 改成"打出 7 盖过 C 的 6"。

### 5.2 例子算到一半断气

例子里一共发生了：
- A 被抽 +1（B 那次）
- A 被抽 +1（C S&S 那次）
- C 吃掉 3（+1）
- D 吃掉 C 的 6-6（+2）

数了半天 **但最后没给每人这一轮的总分**，只有一句 `L358-360` 抽象总结"结束时各人按…结算"。暴躁新手：**"你把我拉到这儿不就是要把一轮算完给我看吗？算到一半断气你在逗我？"**

## §7 二刷建议清单（点到即可，不做实施）

1. **RoundExample 挪到最前 + 默认展开**（让新手先看例子建立直觉）
2. **加一个"术语小词典"锚点区**：局/轮/回合、场上/你的得分堆、盖过、翻面（三种）等核心术语一次说清
3. **三选一胶囊与详细段二选一**：要么胶囊做锚点导航、要么直接砍掉
4. **翻面机制合并成一个"🔁 翻面总览"区块**：一次讲清"单张翻面 / Scout 时翻面 / 整副翻面"三种场景
5. **统一位置名词**：全局只用"场上牌组 / 你的得分堆"两个词，其余词替换
6. **计分胶囊加 +/− 色差**：绿 / 红；公式与胶囊术语对齐
7. **底部按钮文案升级**：`知道了` → `我懂了，开打！`
8. **修 RoundExample 硬 bug**：6 / 6-6 二选一、例子末尾给出每人总分
9. **强度比较图加递进视觉**：左端"最强"标签 + 渐弱色度 / 或 ① ② ③ ④ 排序
10. **Show 子规则和 Scout / S&S 层级对齐**：要么 Scout / S&S 也开子框，要么把"强度比较"提级成平级段落

## 🗯 斗地主老哥二刷拍桌总结

> **"v1 的十条我是满意的，你态度端正。但是你这一页现在是糊到我眼上的十层糖纸——色块也加了、胶囊也加了、折叠卡也加了，但我还是得读两遍、跳三处、算一半。**
> **暴躁新手学新游戏只要三样东西：① 一个完整例子先看懂大概；② 一个术语词典随时查；③ 一张参考图随时翻。你现在三样都有，但全被藏在奇奇怪怪的位置——该前置的后置，该合并的打散，该默认展开的折叠。下次能不能问问我再排版？"**
